# 알려진 이슈: 창 이동 후 포커스 유실

## 1. 현상
단축키로 창을 이동/크기 조절한 직후, 연달아 단축키를 입력하면 반응 없음.
창을 마우스로 클릭하여 포커스를 다시 잡으면 정상 작동.
스마트 사이클 연속 입력이 불가능해지는 증상.

## 2. 원인 분석
- AX API로 창의 위치/크기를 변경한 후, macOS가 해당 창의 포커스 상태를 일시적으로 해제할 수 있음
- Carbon 단축키 리스너에서 이벤트 처리 후 현재 키 상태를 초기화하는 과정에서 수식 키(Cmd, Ctrl 등) 상태가 함께 사라져 후속 입력이 무시될 수 있음

## 3. 해결 방안
- `setWindowFrame()` 호출 직후, 해당 앱을 명시적으로 최상위로 올리고 포커스 재부여
  - CGo에서 `AXUIElementSetAttributeValue`로 `kAXFrontmostAttribute` 설정
- Carbon 이벤트 핸들러에서 키 상태 초기화 시 수식 키는 유지하도록 처리

## 4. 검증 방법
1. 임의의 창(예: Chrome)을 활성화
2. 좌측 절반 단축키 입력 → 창 이동 확인
3. 즉시 동일 단축키 재입력 → 1/3으로 스마트 사이클 동작 확인 (포커스 유실 없어야 함)
