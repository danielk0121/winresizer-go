# 알려진 이슈: 창 최소 크기 제한으로 인한 우측/하단 화면 짤림

## 1. 현상
커스텀 비율(예: 35%)이 앱(크롬 등)의 최소 폭 제한보다 작을 경우:
- **좌측 정렬**: 정상처럼 보임 (x=0 기준이라 창이 오른쪽으로 더 넓어질 뿐)
- **우측 정렬**: 창이 오른쪽 화면 밖으로 짤림
- **스마트 사이클**: 짤린 상태에서 다음 단계로 넘어가지 못하고 무한 반복

## 2. 원인
- `CoordinateCalculator`는 앱의 최소 크기를 모른 채 `x = 모니터너비 × (1 - 비율)`로 계산
- 예: 모니터 1000px, 35% 설정 → x=650, w=350 요청
- 크롬 최소 폭 500px → 실제로는 `x=650, w=500` 적용 → 오른쪽 끝이 1150px (모니터 초과)
- 사이클 판정 로직이 짤린 상태를 "아직 정렬 안됨"으로 판단하여 계속 같은 동작 반복

## 3. 해결 방안: Re-anchoring (좌표 재보정)
`setWindowFrame()` 호출 후 실제 적용된 창 크기를 다시 읽어서 좌표를 재보정한다.

```
// 우측 정렬 시 재보정 예시
actualBounds = getWindowFrame(pid)
if actualBounds.width > targetWidth {
    // 실제 창이 더 넓으면 오른쪽 엣지를 모니터 우측에 맞춤
    correctedX = monitorRight - actualBounds.width
    setWindowPosition(pid, correctedX, actualBounds.y)
}
```

사이클 판정도 보정된 좌표 기준으로 수행하여 짤린 상태에서도 다음 단계로 진행 가능하도록 처리.

## 4. 검증 방법
- 크롬 브라우저에 커스텀 35% 우측 정렬 적용 후 화면 짤림 없는지 확인
- 짤린 상태에서도 스마트 사이클이 다음 단계로 넘어가는지 확인
